## 1. Overview

### 1.1 Purpose

모바일 장치에 최적화된 웹 레이아웃을 구현하여 모든 화면 크기에서 일관된 사용자 경험을 제공합니다. 모바일 퍼스트 접근법을 통해 모바일 사용자에게 최적의 성능과 UX를 보장하면서도 태블릿과 데스크톱에서 점진적으로 향상된 경험을 제공합니다.

### 1.2 Design Philosophy

- **모바일 퍼스트**: 320px 모바일 화면부터 시작하여 점진적 향상(Progressive Enhancement)
- **접근성 우선**: 키보드 내비게이션, 스크린 리더 호환성, 색상 대비 충분
- **성능 중심**: 이미지 최적화, CSS/JS 최소화, 빠른 로딩 속도
- **유지보수성**: 명확한 폴더 구조, 재사용 가능한 컴포넌트

### 1.3 Architecture Summary

실용적 균형(옵션 C) 아키텍처를 선택했습니다:
- **장점**: 학습 용이성과 실용성 균형, 명확한 관심사 분리
- **구조**: 모듈화된 CSS, 컴포넌트 기반 JS, 반응형 HTML
- **도구**: 순수 HTML/CSS/JS (프레임워크 없음)

## 2. Architecture Decisions

### 2.1 Framework Selection

| Option | Selected | Rationale |
|--------|:--------:|-----------|
| React/Next.js | ☐ | Starter 레벨, 학습 목적에 불필요한 복잡성 |
| Vue/Nuxt | ☐ | 프레임워크 학습 부담 |
| **HTML/CSS/JS** | ✅ | 기본 개념 학습, 가볍고 빠른 구현 |

```

├── index.html              # 메인 페이지
```

## 3. Data Model

### 3.1 CSS Design Tokens (변수 시스템)

```css
/* css/base/variables.css */
:root {
  /* 색상 */
  --color-primary: #3b82f6;
  --color-secondary: #10b981;
  --color-text: #1f2937;
  --color-background: #ffffff;
  
  /* 타이포그래피 */
  --font-family-base: 'Inter', '맑은 고딕', sans-serif;
  --font-size-xs: 0.75rem;   /* 12px */
  --font-size-sm: 0.875rem;  /* 14px */
  --font-size-base: 1rem;    /* 16px */
  --font-size-lg: 1.125rem;  /* 18px */
  --font-size-xl: 1.25rem;   /* 20px */
  
  /* 간격 */
  --spacing-xs: 0.25rem;  /* 4px */
  --spacing-sm: 0.5rem;   /* 8px */
  --spacing-md: 1rem;     /* 16px */
  --spacing-lg: 1.5rem;   /* 24px */
  --spacing-xl: 2rem;     /* 32px */
  
  /* 브레이크포인트 */
  --breakpoint-mobile: 320px;
  --breakpoint-tablet: 768px;
  --breakpoint-desktop: 1024px;
}
```

### 3.2 Responsive Breakpoints

| Breakpoint | Device | CSS Media Query | Purpose |
|------------|--------|-----------------|---------|
| Mobile | 모바일 | `@media (min-width: 320px)` | 기본 스타일 (모바일 퍼스트) |
| Tablet | 태블릿 | `@media (min-width: 768px)` | 태블릿 레이아웃 향상 |
| Desktop | 데스크톱 | `@media (min-width: 1024px)` | 데스크톱 최적화 |

## 5. UI/UX Design

### 5.1 Mobile-First Layout

**모바일 (320px-767px):**
- 단일 열 레이아웃
- 햄버거 메뉴 네비게이션
- 터치 친화적 버튼 (최소 44×44px)
- 간결한 콘텐츠 계층 구조

**태블릿 (768px-1023px):**
- 2열 그리드 시스템
- 확장된 네비게이션 (햄버거 제거)
- 더 큰 타이포그래피
- 이미지 최적화

**데스크톱 (1024px+):**
- 3-4열 그리드 시스템
- 전체 네비게이션 표시
- 세부 정보 표시
- 호버 효과 추가

### 5.2 Accessibility Features

- **키보드 내비게이션**: `tabindex`, `:focus-visible`
- **스크린 리더**: ARIA 라벨, 의미론적 HTML
- **색상 대비**: WCAG AA 기준 (4.5:1)
- **이미지 대체 텍스트**: 모든 이미지에 `alt` 속성
- **폼 접근성**: `label`과 `input` 연결

### 5.3 Touch Interaction

| Element | Size | Spacing | Purpose |
|---------|------|---------|---------|
| 버튼 | 44×44px | 8px 간격 | 터치 정확도 보장 |
| 링크 | 최소 44px 높이 | - | 손가락으로 쉽게 탭 |
| 폼 입력 | 44px 높이 | 16px 라벨 간격 | 편리한 입력 |

## 9. Risks and Mitigation

### 9.1 Technical Risks

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| 브라우저 호환성 | Medium | Low | Progressive Enhancement, Feature Queries |
| 성능 저하 (모바일) | High | Medium | 이미지 최적화, 코드 스플리팅 |
| 접근성 미준수 | High | Medium | 접근성 검사 도구 통합 |
| CSS 복잡도 증가 | Medium | Medium | CSS 모듈화, BEM 명명법 |

### 9.2 User Experience Risks

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| 모바일 네비게이션 불편 | High | Low | 사용성 테스트, 터치 친화적 디자인 |
| 콘텐츠 가독성 문제 | Medium | Medium | 타이포그래피 시스템, 충분한 대비 |
| 로딩 속도 저하 | High | Medium | 이미지 지연 로딩, CSS 최소화 |
