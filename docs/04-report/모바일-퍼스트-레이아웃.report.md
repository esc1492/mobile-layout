## 6. Lessons Learned & Retrospective

### 6.1 What Went Well (Keep)

- **체계적인 PDCA 프로세스**: Plan → Design → Do → Check → Act 흐름이 명확한 구현 가이드 제공
- **디자인 참조 주석**: 모든 주요 파일에 `Design Ref: §{section}` 주석으로 코드-설계 추적성 확보
- **모듈화된 구현**: 4개 모듈(기본 구조, 반응형 그리드, 네비게이션, 최적화)로 분할 구현 효율성
- **접근성 우선 설계**: ARIA 속성, 키보드 내비게이션, 스크린 리더 지원 초기부터 고려

### 6.2 What Needs Improvement (Problem)

- **초기 범위 정의**: 예시 페이지(`about.html`, `contact.html`) 범위 명시 부족
- **테스트 계획 통합**: 구현 중 테스트 계획 통합 부족 (사후 검증으로 이관)
- **문서 동기화**: 구현 중 디자인 문서 실시간 업데이트 미흡
- `console.log()` **관리**: 개발 중 디버깅 코드 프로덕션에서 자동 제거 메커니즘 필요

### 6.3 What to Try Next (Try)

- **테스트 주도 개발(TDD) 접근법**: 구현 전 테스트 케이스 작성 검토
- **자동화된 코드 검사**: Pre-commit 훅으로 `console.log()` 자동 제거
- **실시간 문서 동기화**: 구현 변경 시 디자인 문서 자동 업데이트 도구 검토
- **성능 기준 통합**: Lighthouse 점수를 Definition of Done에 포함

## Appendix A: Technical Implementation Details

### A.1 CSS Architecture
- **모바일 퍼스트 미디어 쿼리**: `@media (min-width: 320px)` 기반
- **확장 브레이크포인트**: 320px, 480px, 640px, 768px, 1024px, 1280px, 1536px
- **CSS 변수 시스템**: 40+ 디자인 토큰 (색상, 타이포그래피, 간격, 브레이크포인트)
- **그리드 시스템**: 12열 CSS Grid + Flexbox 폴백

### A.2 JavaScript Components
- **네비게이션**: 햄버거 메뉴 토글, 키보드 내비게이션, ESC 키 지원
- **이미지 로딩**: Intersection Observer API + 폴백 구현
- **접근성**: ARIA 라벨 동적 업데이트, 포커스 트랩 관리

### A.3 Performance Optimizations
- **이미지 최적화**: `data-srcset`, `sizes` 속성, WebP 폴백
- **CSS 최소화**: Critical CSS 분리, 미사용 코드 제거
- **JavaScript 지연 로딩**: `defer` 속성, 모듈화 번들

### A.4 Accessibility Features
